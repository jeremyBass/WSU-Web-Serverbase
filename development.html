WSU Serverbase development guide

What is this for?

This project is used in aidding your development and web design taskes. To achive this result of a simple get up and running to start your real work we use Vagrant along with a Salt provisioner to set up the local server and deploy the web apps.

Vagrant options

At the top of the Vagrantfile, which is a ruby coded file, you have some options that you may use to change your environment that you are loading.

#######################
# CONFIG Values
#####################
ip="10.10.30.30"        # (str) default:10.10.30.30             - ip of the VirturalBox
hostname="WSUBASE"      # (str) default:WSUBASE                 - name of the host
memory=512              # (int) default:512                     - How much memory would you like to share from your host
cores=2                 # (int) default:1                       - How many processor from the host do you want to share
host_64bit=true         # (bool) default:false                  - If you are on windows and are sharing more then 2 cores set to true
install_type='testing'  # (testing) default:testing             - Type of install
minion='vagrant'        # (vagrant/production) default:vagrant  - Which minion to run
verbose_output=true     # (bool) default:true                   - How much do you want to see in the console

How to add a web app?

Loading a web app is very straight forward. You simply git clone in the app of choice into the app/ folder at the root for the server folder. You may load as many apps as you wish to, but it should be noted if you need to simulate one server talking with another, then you will want to have two instances of this project and load the web app respectively.
