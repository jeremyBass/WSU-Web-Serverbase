WSU Serverbase

Github : WSU-Web-Serverbase

Overview

This is a project that is meant to create a server environment in one of two locations. One is with a Vagrant controlling a headless VirtualBox, and the other is on a Centos 6.0+ server. You can look through the WSU-Web-Serverbase repo for details but this mirco site is meant to give you a simple overview of how to use and what is done. This project may be loaded with web apps such as Magento,Elasticsearch,Wordpress, or any other app that has be set up to be provisioned. To get the LEMP setup running for the first time, there are only a few step needed.

For Local deveploment

  1. Install required applications:
    1. Git (Windows or OSX) More infomation on Git and other client downloads
    2. Vagrant
    3. VirtualBox
    NOTE: There has been versions of Vagrant and VirtualBox that do play well with each other. We will try to keep up a list of combinations that had been noted to not work.
  2. Clone the WSU Web Serverbase repository to a directory on your local machine. 
    $ git clone git@github.com:washingtonstateuniversity/WSU-Web-Serverbase.git wsuweb
    Note:This can be a powershell/command line/terminal depending on what is installed on you system.

  3. At the command line, change to the new directory:

    $>_ cd wsuweb

  4. (optional) Add web apps to the server. Example:

    $>_ git clone git@github.com:washingtonstateuniversity/WSUMAGE-base.git app/store.wsu.edu

  5. Start the local server:

    $>_ vagrant up
    NOTE: If you would like to do some logging of Vagrant setup, you can use $>_ vagrant up | tee log.txt instead. There will be a log of the whole run located at wsuweb/log.txt

Those are the steps for your local dev environment. You may also highstate the project on its own, but it must be on a ready server. That is salt-call state.highstate env=base.
